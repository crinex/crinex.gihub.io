Vision Mamba와 ECG를 활용한
심장 질환 환자 분류

Da-Woon Kwon1 , Seo-Yeon Oh2, Ye-Hyun Ko3,
1University of Ulsan, 2Seoul Women's University, 3Soongsil University

Abstract

본 프로젝트는 기존 Transformer 모델의 한계를 극복하기 위한 'Vision Mamba' 아키텍처를 소개하고, 이를 심전도(ECG) 데이터에 적용하여 심장 질환 환자를 classification한다. Transformer는 많은 수의 파라미터를 가지고 있으며, Self-Attention이 시간 복잡도 면에서 비효율적이라는 문제가 있다. 따라서 SSM을 통해 ‘state’라는 개념을 도입하는 새로운 recurrent 메커니즘 제안하고, S4를 통해 이산화(Discretization, ZOH)와 HiPPO 매트릭스를 활용한 AI 모델을 개발한다. 다음으로 S6을 통해 Mamba의 핵심인 Selective Scan 알고리즘을 소개하며, 이를 Gated NLP와 H3와 결합하여 Mamba 블록을 구성, Mamba 아키텍처를 완성한다. 우리의 연구 핵심은 연속적 특성을 가진 ECG 데이터를 Vision Mamba 아키텍처에 적용하여 분류하는 것이다. 이미지 데이터를 시퀀스 임베딩으로 변환하고, 이미지를 패치로 분할하여 patch와 position 임베딩을 결합한 최종 input 임베딩을 생성한다. 12개 채널의 ECG 데이터를 이산 벡터로 표현하고 연결하여, 시간 단계에 맞춰 패치로 분할함으로써 Vision Mamba의 input으로 사용하여 학습한다. 우리의 결과 및 추후 분석은 Vision Mamba가 ECG 데이터를 행렬(이미지) 관점에서 해석하는 능력을 보여주고, Mamba 아키텍처가 AI 분야에서 주목할만한 발전이 될 수 있음을 강조하고자 한다.

Transformer의 문제점

1. 파라미터의 수가 많다
파라미터 수를 줄이는 것은 파라미터를 훼손하는 것이기 때문에 파라미터가 표현하는 수의 범위를 줄이는 양자화(Quantization) 연구가 진행되어 오고 있다. 양자화는 예를 들어 32bit 범위의 파라미터를 더 적은 1bit로 표현하여 모델의 성능을 훼손시키지 않는 선에서의 기술을 의미한다.
2. self-Attention의 비효율적인 시간복잡도
6개의 시퀀스를 가진 하나의 토큰이 있다고 가정할 때 자신과 똑같은 자식의 시퀀스에 대해 각 토큰이 시퀀스의 길이만큼 탐색을 하며 어떤 토큰이 중요한 정보를 담고 있는지 판단한다. 이러한 과정에서 O(n^2) 만큼의 복잡도가 발생한다. 즉, inference를 수행하면 너무 비효율적이다.

Transformer은 기존의 recurrent 메커니즘을 해결하는 기술이었고, 작년 11월 새로운 recurrent 메커니즘을 사용하는 Mamba가 공개되었다. Mamba는 아래 표와 같이 transformer보다 엄청나게 빠른 inference를 수행하며, performance 측면에서도 transformer에 준하거나 오히려 뛰어넘는 것을 확인할 수 있다.

SSM (State Space Model)

상태 공간 모델은 시간에 따라 변하는 과정을 모델링하기 위한 프레임워크로 시스템의 ‘상태’와 이 상태가 시간에 따라 어떻게 변화하는지를 기술한다. 일반적인 모델의 경우 input output에 대해 고려한다면, ssm은 state라는 개념 도입하여 input, output, state에 대해 고려한다. 이 때, state는 어떠한 system을 설명하기 위한 최소 개수의 변수 집합이다.

SSM의 수식 표현

discrete하게 표현했을 때는 현재상태가 이전상태에 의존한다는 것에서 RNN과 유사함을 알 수 있다. RNN과 다른 점은 각 방정식에 대해서 A,B,C,D 행렬이 곱해져 있다는 것이다.

문제점
1. SSM은 주로 continous한 데이터에 최적화되어 있기 때문에, discrete 데이터를 효과적으로 다루기 위해서는 discretization이 필요하다.
2. 길어진 시퀀스에 대해서 정보를 잘 담아내지 못하여 long dependency 처리에 어려움이 있다.


S4 (Structured State Spaces for Sequences)

SSM 문제에 대한 Solution

1. ZOH(Zero Order Hold) 도입
 각 데이터 포인트를 다음 데이터 포인트가 나타날 때까지 유지하면서 연속적으로 표현하는 방식이다.
⇒ discrete 데이터를 연속적인 형태로 자연스럽게 변환하며, SSM이 discrete 데이터를 처리하는 데 있어서 발생하는 문제를 해결한다.



2. HiPPO (History and Prediction in Polynomial time) 매트릭스
긴 시간 동안의 의존성 문제를 해결하며, 이전 상태를 잘 기억하기 위해서 중요한 A matrix를 HiPPO 매트릭스로 초기화함으로써, SSM의 긴 시퀀스 처리 문제를 해결한다.



S6 (Selective Scan Structured State Spaces for Sequences)


A matrix : HiPPO matrix로 state transmission 역할을 수행하기 때문에 제외한다.
고정된 크기의 B,C,D 행렬 : 시퀀스 벡터처럼 Batch Size, Sequence length만큼 행렬을 텐서로 만들면 Selective하게 정보를 선택 가능하다.


S6를 블록으로 사용해서 Gated NLP와 H3를 합쳐서 mamba block으로 사용함으로써 Mamba라는 아키텍처 제안하고 있다.

Vision Mamba

우리가 프로젝트에 사용하는건 Vision Mamba이지만, Mamba나 Transformer은 sequential data를 다루지 않았나?
따라서, 모델에 넣을 image데이터를 sequential한 embedding으로 바꾼다.


4X4 이미지가 있다고 할 때, 16개의 patch로 분할하여 projection과 flatten을 통하여 하나의 embedding으로 만들어주고, 이미지는 각 패치가 어느위치에 있었느냐가 중요한 정보이기에 position matrix 또한 새로운 embedding으로 바꾼다. 두 embedding을 더하여 vision mamba에 넣을 input으로 바꿔준다.


ECG 데이터

ECG란 심전도 검사로 심장의 운동을 전극으로 측정하여, signal data로 기록한 의료 데이터의 한 종류이다.


METHOD

ECG 처리 관련 논문 중 한 lead를 vector로 보는 관점에서 12개의 lead를 전부 concatenate 시켜 하나의 이미지로 처리한다. 이 이미지를 time step(1초)에 따라 patch로 나누어 vision mamba의 input으로 사용하여 학습시키는 방식을 채택하여 Vision Mamba를 프로젝트에 사용했다.
